Lesson 1 — Data Structures & Algorithms in JavaScript

Welcome to Data Structures

00 — The Big Picture

What is "DSA," really?

DSA stands for Data Structures and Algorithms. It's not a JavaScript thing, or a Python thing — it's a general computer science idea. Every programming language lets you organize data (data structures) and write steps to solve problems (algorithms). We'll learn the ideas using JavaScript, but the ideas themselves work everywhere.

In one sentence Data structures are the containers for information, and algorithms are the instructions for doing something useful with it. Learning DSA means learning to pick good containers and write good instructions.

Why bother learning this?

You can write code without ever thinking about this. But as soon as an app has a lot of data — thousands of users, millions of posts, a big product catalog — the way that data is organized decides whether the app feels instant or feels frozen. Learning DSA teaches you to:

Write faster code

Know which container and which steps keep your app quick, even with lots of data.

Solve problems, not just syntax

Break a big, messy problem into small, clear steps — a skill that outlasts any one language.

Read other people's code

Most real codebases are built on these same basic ideas, so you'll recognize patterns faster.

Do well in interviews

DSA is one of the most common topics companies ask about when hiring developers.

Where do you already see this?

DSA isn't abstract — you use apps built on it every day:

Maps & GPS

Finding the fastest route uses graphs (roads and intersections) and pathfinding algorithms.

Autocomplete & search

Suggesting words or search results fast, even with millions of options, relies on smart data structures.

Undo button

Ctrl+Z works like a stack — the last thing you did is the first thing that gets undone.

Music queues & playlists

"Play next" behaves like a queue — songs play in the order they were added.

Same ideas, different accents

Every language has its own way of writing these ideas — a small difference in syntax, not in concept. A JavaScript array, a Python list, and a Java array are all "a row of numbered lockers" underneath. Once you understand the idea in JavaScript, you can recognize it instantly in almost any other language — only the exact words and punctuation change.

01 — Key Words

A few words we'll use a lot

Before anything else, let's put simple meanings on some words you'll hear all the time in this course. You don't need to memorize these — just get a rough feel for them. We'll come back to each one.

Data

Any piece of information a program works with — a name, a number, a score, a list of friends.

Data structure

A way of arranging data so it's easy to use. Like choosing a drawer, a box, or a shelf for your stuff.

Algorithm

A list of steps to solve a problem — like a recipe. We'll write our first one later this course.

Memory (RAM)

The part of a computer that holds data while a program is running. Think of it as storage space.

Memory address

A number that marks exactly where one piece of data lives in memory — like a house number on a street.

Index

A number that marks a position inside a data structure like an array. Counting starts at 0 in JavaScript.

Time complexity

A rough measure of how long something takes as the amount of data grows.

Space complexity

A rough measure of how much extra memory something needs as the amount of data grows.

Big O

The short way programmers write time or space complexity, like O(n). We'll unpack this below.

02 — Data Structures

Why not just dump everything in one pile?

A data structure is a way of organizing data so it's easy to find, add, or remove things later. In this course we'll organize and use data with JavaScript. Different jobs need different kinds of organization.

Think of it like... Your bedroom. Throw everything into one pile, and finding your headphones takes forever. Give clothes a drawer, books a shelf, and cables a small box — now you know exactly where to look. Data structures are the drawers, shelves, and boxes for your data.

Here are a few common data structures. We'll use JavaScript arrays a lot this course, and meet the others later on.

Lists / Arrays

Items lined up in order, like socks in numbered drawer slots. This is our focus today.

Stacks

Like a stack of plates — you add and remove from the top.

Queues

Like a line at the movies — first person in, first person served.

Trees & Graphs

Like a family tree or a subway map — things connected to other things.

Data structures + algorithms: a team

A data structure organizes the "stuff." An algorithm is the steps that do something with that stuff — search it, sort it, combine it. Pick the wrong container and even a great algorithm gets slow. We'll write our first algorithm together soon — for now, just remember the two always work as a pair.

03 — Memory

Every piece of data has an address

A computer's memory (RAM) is like a long street of small houses. Every house holds one piece of data, and every house has its own address — a number.

When your JavaScript code creates a variable, like let name = "Sam", the computer finds an empty house in memory and writes the value there. The computer remembers the address, not the value itself — like remembering a street number instead of memorizing what's inside the house.

This matters because how a data structure arranges its houses changes how fast we can find things later. Some structures (like arrays) keep houses in a neat row, so it's fast to jump around. We'll see why that makes arrays quick to work with.

04 — Complexity

How much slower does this get as data grows?

This is what programmers call Big O. Don't worry about the funny symbol — it's just a rough label for how long something takes as the pile of data grows.

Think of it like... Looking for a name in a phone book. Know the exact page? Flip straight there — fast, no matter how thick the book is. No idea where to look? You might check every page — slow, and worse as the book gets thicker. Big O describes which situation you're in.
O(1)
Instant — grabbing item #3 directly
O(log n)
Quick — cutting the search in half each time
O(n)
Fair — checking every item once
O(n²)
Slow — comparing every item to every other item

There's a related idea called space complexity. Instead of asking "how long does this take?", it asks "how many extra memory houses does this need?" A fast approach that uses a lot of extra memory isn't always the best choice.

05 — Arrays

The array: your first data structure in JavaScript

An array is a row of numbered lockers sitting next to each other in memory. Each locker holds one value, and each locker has a number — its index.

Important habit JavaScript (like almost every language) starts counting at 0, not 1. So fruits[0] is the first locker, and fruits[4] is the last one in a 5-item array. This trips up everyone at first — you're in good company.

What can you do with an array?

OperationWhat it meansJavaScript
AccessLook at what's in a specific lockerfruits[2]
Add to the endPut a new item in the next open lockerfruits.push("peach")
Remove from the endTake the last item outfruits.pop()
Add to the frontSlide everyone down, insert at locker 0fruits.unshift("mango")
Remove from the frontTake out locker 0, slide everyone upfruits.shift()
Find lengthCount how many lockers are filledfruits.length

Adding or removing at the end is quick, but at the front every other item has to shuffle down one locker — more work as the array grows.

Recap before we build anything

Data structures organize information • memory stores everything at numbered addresses • Big O describes how speed changes as data grows • arrays are ordered, zero-indexed lockers. Next up in class: putting arrays to work, and building our first algorithm together.
